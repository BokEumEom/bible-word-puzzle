## 진단: 현재 가장 큰 갭

**듀오링고에는 학습 경로(Tree)가 있고, 말씀 팡팡에는 없다.**

현재 앱은:
- 프리셋 24개 구절만 큐레이션 (66권 데이터는 자유 탐색 전용)
- 성경 브라우저는 구약/신약 평면 리스트
- "내가 얼마나 했는지" 시각적 진행감 부재
- 추천 엔진은 있지만 여정(journey) 없음

## 솔루션: 테마별 컬렉션 시스템

### 구조

```
홈 화면
├── 오늘의 말씀 (기존 유지)
├── 📖 테마 컬렉션 (NEW — 메인 학습 경로)
│   ├── 사랑의 말씀  ██████░░ 6/10
│   ├── 소망의 말씀  ████░░░░ 4/8
│   ├── 힘이 되는 말씀 🔒 잠금
│   ├── 위로의 말씀   🔒 잠금
│   ├── 지혜의 말씀   🔒 잠금
│   ├── 감사의 말씀   🔒 잠금
│   ├── 기도의 말씀   🔒 잠금
│   └── 믿음의 말씀   🔒 잠금
├── CTA 버튼 (기존 유지)
└── 최근/즐겨찾기 (기존 유지)
```

### 핵심 설계 원칙

1. **잠금 해제 순서**: 첫 2개 컬렉션 열림 → 50% 완료 시 다음 해제
2. **난이도 점진**: 각 컬렉션 내 구절이 쉬움→어려움 순서
3. **컬렉션 = 코스**: 듀오링고의 Unit과 동일한 역할
4. **자유 탐색은 유지**: 기존 "성경 찾기" 그대로 보존
5. **기존 데이터 활용**: 66권 JSON에서 구절 매핑

### 임팩트 예측

- **리텐션**: "다음 구절"이 생김 → 재방문 동기
- **진행감**: 프로그레스 바가 채워지는 시각적 보상
- **발견**: 테마별로 다양한 성경 구절 노출
- **온보딩 연계**: 관심 테마 기반 첫 컬렉션 추천

## Quick Wins (이 Phase에서)

1. 테마 컬렉션 데이터 정의 (~8개 테마, 각 6-10 구절)
2. 컬렉션 리스트 UI (프로그레스 바 + 잠금 상태)
3. 컬렉션 상세 화면 (구절 목록 + 순서대로 풀기)
4. Dashboard에 컬렉션 섹션 추가
5. 잠금 해제 로직 + 보상 연출
